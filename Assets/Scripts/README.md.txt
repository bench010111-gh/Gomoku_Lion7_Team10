# Scripts 폴더 구조

이 폴더는 프로젝트에서 사용하는 스크립트들을 기능 기준으로 분류한 폴더입니다.  
각 폴더는 담당 기능이나 게임 모드에 따라 나뉘어 있으며, 새로운 스크립트를 추가할 때도 해당 역할에 맞는 폴더에 넣는 것을 원칙으로 합니다.

---

## 01_Core
프로젝트 전체에서 공통으로 사용되는 핵심 스크립트를 모아둔 폴더입니다.  
특정 모드에 종속되지 않고, 게임 전반의 기본 흐름이나 공용 세션 유지와 관련된 스크립트가 들어갑니다.

예시:
- `StartSceneManager.cs`
- `UserSession.cs`

---

## 02_Backend
뒤끝 연동과 관련된 스크립트를 모아둔 폴더입니다.  
회원가입, 로그인, 유저 데이터 생성/조회/저장 등 뒤끝 서버와 통신하는 기능이 포함됩니다.

예시:
- `BackendManager.cs`
- `BackendAuthManager.cs`
- `PlayerDataService.cs`

---

## 03_MainLobby
메인 로비와 관련된 스크립트를 모아둔 폴더입니다.  
로그인 이후 메인 로비에서 모드를 선택하거나, 플레이어 정보 및 전적을 표시하는 기능 등이 포함됩니다.

예시:
- `MainLobbyManager.cs`
- `MainLobbyPlayerInfoUI.cs`

---

## 04_Multi
멀티 모드 선택 이후부터 사용되는 모든 스크립트를 모아둔 폴더입니다.  
멀티 로비, Photon 방 생성/입장, 멀티 게임 씬 이동, 채팅, 멀티 동기화 등 멀티 플레이와 직접적으로 관련된 기능을 담당합니다.

예시:
- `PhotonLobbyManager.cs`
- `PhotonChatManager.cs`
- `MultiGameBackHandler.cs`

---

## Single
싱글 모드 선택 이후 사용되는 스크립트를 모아둔 폴더입니다.  
스토리 진행, AI 대전, 싱글 전용 게임 흐름과 관련된 기능이 포함됩니다.

예시:
- 추후 싱글 모드 진행 스크립트
- AI 연결 스크립트
- 스토리/스테이지 관련 스크립트

---

## Duo
듀오(한 기기 2인) 모드 선택 이후 사용되는 스크립트를 모아둔 폴더입니다.  
같은 기기에서 두 플레이어가 번갈아 진행하는 게임 흐름과 관련된 기능이 포함됩니다.

예시:
- 추후 듀오 모드 진행 스크립트
- 듀오용 게임 매니저

---

## Data
프로젝트에서 공통으로 사용하는 데이터 구조를 모아둔 폴더입니다.  
오목판 상태, 착수 기록, 전적 구조 등 규칙 처리나 저장에 필요한 데이터 클래스가 들어갑니다.

예시:
- `BoardData.cs`
- `MoveData.cs` (추후 분리 시)
- 기타 공통 데이터 클래스

---

## Rules
오목 게임의 공통 규칙과 관련된 스크립트를 모아둔 폴더입니다.  
모드와 관계없이 공통으로 사용되는 판정 로직을 여기에 둡니다.

예시:
- 착수 가능 여부 검사
- 금수(렌주룰) 검사
- 승패 판정
- 무승부 판정

주의:
- `Rules` 폴더의 스크립트는 싱글 / 듀오 / 멀티가 공통으로 참조할 수 있도록 작성합니다.

---

## UI
UI 담당자가 사용하는 스크립트를 모아둘 폴더입니다.  
공통 UI, 팝업, 버튼 동작, 텍스트 표시 등 UI 관련 기능이 포함됩니다.

예시:
- 설정 UI
- 결과 팝업 UI
- 공통 버튼/패널 UI 처리 스크립트

---

## Etc
기타 편의 기능이나 분류가 애매한 스크립트를 모아두는 폴더입니다.  
임시 유틸리티 성격이 강하거나, 특정 기능에 바로 속하지 않는 스크립트를 여기에 둡니다.

예시:
- `BackButtonHandler.cs`
- 기타 공통 편의 기능 스크립트

주의:
- `Etc` 폴더는 너무 많은 스크립트가 몰리지 않도록 주의하고, 역할이 명확해지면 가능한 적절한 폴더로 이동하는 것을 권장합니다.

---

# 폴더 사용 원칙

1. 새로운 스크립트를 만들 때는 먼저 이 스크립트가 어느 기능/모드에 속하는지 확인한 뒤 해당 폴더에 넣습니다.
2. 공통 데이터와 공통 규칙은 가능한 `Data`, `Rules` 폴더에 분리합니다.
3. 모드 전용 흐름 스크립트는 `Single`, `Duo`, `Multi` 폴더에 각각 분류합니다.
4. 분류가 애매한 경우 일단 `Etc`에 두되, 추후 구조가 정리되면 적절한 폴더로 이동합니다.
5. 공용 스크립트나 폴더 구조를 변경할 때는 팀원들과 먼저 공유하는 것을 권장합니다.
